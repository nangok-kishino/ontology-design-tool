# ClaudeCode 実装指示書
## オントロジー設計支援ツール

---

## 目的

Azure無料枠での公開を最終目標としてシステムを構築する。

---

## インフラ構成

全サービスをAzureで完結させる。

- **Azure Static Web Apps（無料枠）**：Next.js App Router のホスティングおよびAzure Functionsによるバックエンドの実行
- **Azure Cosmos DB（無料枠）**：データの永続化

ローカル開発環境でも同じ構成を前提として実装すること。

---

## 開発方針

v0.devで生成したNext.js + Tailwind CSS + shadcn/uiのコードをベースに機能を実装する。
1ステップずつ実装・動作確認を行い、確認が取れたらGitコミットして次のステップに進む。
ただし、ここで提示しているステップはあくまで参考に留め、実装を進めていく中で最適化していく。

---

## 画面構成

```
全体
  ダッシュボード
  文書取込み

オントロジー設計

インスタンス管理
  登録済みインスタンス（※v0.devで作成した画面ではクラス用インスタンスとなっているが変更する）
```

## ステップ1：環境セットアップとDB構築・認証設定

Azure Cosmos DBに接続し、データを永続化できる環境を構築してほしい。管理するデータは以下の通り。

- クラス（名前、説明、親クラスへの参照）
- リレーション（名前、説明、始点クラス、終点クラス、親リレーションへの参照）
- 属性（名前、データ型、必須／任意、共通／固有、紐づくクラスまたはリレーション）
- インスタンス（名前、紐づくクラス、登録者、登録日）
- LLM候補（名前、種別、説明、ステータス、審査者、審査日）

複数人が同時に使用することを前提とする。

あわせて、Azure Static Web Appsの組み込み認証機能を使い、Microsoft Entra IDでログインしないとアプリにアクセスできないようにしてほしい。staticwebapp.config.jsonで全ルートに認証を要求する設定を行うこと。

**確認ポイント：Cosmos DBに接続できること、データの読み書きができること。未ログイン状態でアクセスするとEntra IDのログイン画面にリダイレクトされること**

## ステップ2：クラスの一覧表示

この時点ではデータの追加・編集・削除は不要。表示だけでよい。

**確認ポイント：サンプルデータを直接DBに入れ、ツリーが正しく表示されること**

**確認ポイント：編集が反映されること、削除後にツリーから消えること**

## ステップ11：ダッシュボードの実装

登録済みのクラス・インスタンス・リレーションを俯瞰できる画面を実装してほしい。

- クラスとそれに紐づくインスタンスの対応をテーブルで表示する
- リレーションを始点クラス・終点クラスとあわせてテーブルで表示する
- OWL/RDFエクスポートボタンを配置する（次のステップで機能実装）

**確認ポイント：実際に登録したデータが正しく表示されること**

---

## ステップ12：OWL/RDFエクスポート

ダッシュボードのエクスポートボタンを押すと、登録済みのクラス・リレーション・属性をOWL/RDF形式（Turtle形式）の.ttlファイルとしてダウンロードできるようにしてほしい。

**確認ポイント：出力された.ttlファイルの内容が正しい構造になっていること**

---

## ステップ13：文書取込み画面（ファイルアップロードと解析）

文書取込み画面で、ファイルをアップロードしてLLMで解析し、クラス候補とリレーション候補を一覧表示できるようにしてほしい。LLMはAnthropic APIを使用すること。

**確認ポイント：ファイルをアップロードすると候補が一覧表示されること**

---

## ステップ14：LLM候補の採否判断とクラス・リレーション管理への反映

文書取込み画面で、SMEが候補ごとに承認・却下を判断できるようにしてほしい。

- クラス候補の承認時：トップレベルクラスかサブクラスかを選択し、親クラスを指定してクラス管理に反映する
- リレーション候補の承認時：始点・終点クラスを指定してリレーション管理に反映する

**確認ポイント：承認した候補がクラス管理・リレーション管理に正しく反映されること**
